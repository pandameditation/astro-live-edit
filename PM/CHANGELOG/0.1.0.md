# v0.1.0 - outerHTML Feature Implementation

**Released:** 2026-02-10

## What Changed

### outerHTML Support for Attribute Editing

The live edit system now captures and saves **the entire HTML tag** (opening tag + content + closing tag) instead of just inner content. This enables users to edit HTML attributes directly in the browser.

**Key improvements:**
- Edit any HTML attribute in browser: `style`, `class`, `id`, `data-*`, `aria-*`, etc.
- Attribute-only changes are now detected and saved
- Entire tag replaced in source files (not just inner content)
- Nested tags and content preserved correctly

### Implementation Details

**Client side (`/public/live-edit.js`):**
- Changed change detection from `innerHTML` to `outerHTML` (lines 9, 15, 18, 32)
- Now tracks full tag state including all attributes

**Server side (`/edit-server/save-server.js`):**
- Renamed `replaceInnerContent` to `replaceOuterContent`
- Updated tag-finding logic to include closing tag boundaries
- Astro file handler now replaces entire tag span

**Markdown/MDX behavior:**
- Smart detection: tags without custom attributes convert to markdown syntax
- Tags with custom attributes (style, class, etc.) stay as HTML
- Preserves proper list indentation and structure

## Breaking Changes

- Old behavior (innerHTML-only editing) completely replaced
- Source files now receive full outerHTML when saving edits

## Verification

Tested scenarios:
- Adding style attributes to lists
- Editing content with attribute changes
- Markdown list indentation preservation
- Mixed HTML/markdown content handling
